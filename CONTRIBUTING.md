Contributing to Thoughts

Thank you for your interest in contributing to Thoughts. We welcome improvements, bug reports, and documentation updates. Please follow these simple steps to make contributions smooth and actionable.

How to contribute

- Issues: Open an issue describing the bug or feature request with steps to reproduce, expected behavior, and environment information.
- Pull Requests: Fork the repo, create a short-lived feature branch, implement your change, and open a PR against `main` with a clear description and linked issue (if any).

Guidelines

- Keep changes small and focused.
- Follow existing code style and patterns in `script.js` and `custom.css`.
- Run `npm run minify` only when preparing production artifacts; PRs may use unminified files for easier review.
- Include screenshots or a short demo GIF for UI changes.

Commit messages

- Use concise, imperative-style commit messages (e.g., "Fix offline fallback for service worker").

Review process

- PRs will be reviewed by maintainers; expect feedback and iterate on suggested changes.
- Once approved, a maintainer will merge the PR.
